---

title: Anis
---

Anis --- это инструмент, помогающий в динамической
верификации на соответствие формальной модели Event-B.
Для пользователя он выглядит как расширение VSCode,
позволяющее

- работать с текстом модели на Event-B
- загружать трассы, собранные внешними мониторами
- проверять трассы на модели
- получать отчеты о покрытии модели

Расширение Anis использует средства и инструменты
языка Python (pytest в качестве инфраструктуры
тестирования) и его расширения для VSCode.

#### Версия 0.4.0-beta.2

- extension: <https://nextcloud.ispras.ru/index.php/s/WaBC9eBmCn3Nbds>

- workspace: <https://nextcloud.ispras.ru/index.php/s/4sW9pdgBQqAgwdy>

Основные изменения:

- Переработаны несущие множества: удалены объекты-контроллеры,
  оставлено только множество значений

- Переработана генерация класса Machine: инициализируются только
  поля-множества (пустыми). Остальные поля (и поля, которые должны
  быть изначально непустыми множествами) нужно проинициализировать
  явно в fixture `m`

- Переработан способ получения элементов несущих
  множеств: имена у них удалены (при необходимости
  нужно сохранить ссылку на элемент), для получения
  нового элемента нужно использовать только функцию
  `carrier_set_item` из модуля `anis.model.expressions`

- Можно помечать тесты пропускаемыми, если не задействован
  определенный уровень модели; для этого нужно
  использовать переменную `machines` из модулей событий;
  она позволяет узнать, какие машины были задействованы
  для составления данного события и какими были
  машины-источники охранных условий

- workspace: исправлен `PicklingError`, возникающий
  при запуске на Python 3.14

- workspace: небольшие исправления внесены в модель
