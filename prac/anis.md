title: Практическое задание
---

Задание выполняется группой (3-4 человек).

Результатом работы является образ виртуальной машины
OVA со всеми инструментами и артефактами.

В случае написания тестов покрытие по модели
не должно быть менее 80% по строками и ветвям.

## Вариант 1. IMA/EVM

Нужно добавить поддержку
[подсистемы проверки целостности IMA/EVM](https://sourceforge.net/p/linux-ima/wiki/Home/) в
систему динамической верификации для Linux.

[Модель Event-B](model.tgz) уже есть
(`DAC`, `DAC_EXT` описывают дискреционный уровень,
 `IMA`, `IMA_EXT` описывают уровень целостности).

[Методика тестирования](test_method.odt) тоже есть
(выделена желтым цветом в этом документе).

Что нужно сделать:

1. Добавить поддержку IMA/EVM в
   [базовую версию монитора](monitoring.tgz) на основе kprobes/kretprobes.

1. Написать тесты на pytest согласно методике тестирования.
   В качестве примера есть [тесты для DAC](testing.tgz).

2. Выполнить тестирование ядра со сбором трасс.

3. Написать медиатор. В проекте с моделью Event-B есть
   медиатор для DAC.

4. Выполнить проверку трасс на модели.

5. Подготовить отчет с оценкой трудоемкости и оценкой
   программной инфраструктуры.
